# 無常損失（Impermanent Loss）

## 定義

**無常損失 = 流動性提供者（LP）因資產價格變動而產生的相對損失**

當你將資產存入流動性池（如 Uniswap）後，如果兩種資產的相對價格發生變化，相比於單純持有（HODL），你的資產價值會減少。這種損失稱為「無常損失」。

```
簡單理解：
存入流動性池的價值 < 單純持有的價值 = 無常損失
```

## 為什麼叫「無常」

- **暫時性**：如果價格回到存入時的比例，損失消失
- **實現條件**：只有在你退出流動性時才會真正實現
- **對比基準**：相對於「單純持有」的機會成本

## 無常損失如何產生

### AMM 的運作機制
[[DEX]] 使用自動做市商（AMM），遵循 **x × y = k** 公式：
- x = 代幣 A 的數量
- y = 代幣 B 的數量
- k = 常數

當價格變動時，池子會自動調整兩種代幣的比例，導致你持有的數量發生變化。

### 範例說明

**初始存入（1 ETH = $1000）：**
| 資產 | 數量 | 價值 |
|------|------|------|
| ETH | 1 | $1,000 |
| USDC | 1,000 | $1,000 |
| **總計** | - | **$2,000** |

**ETH 漲價 100%（1 ETH = $2000）後：**

由於 AMM 機制，池子會賣出 ETH、買入 USDC 來維持平衡：
| 資產 | 數量 | 價值 |
|------|------|------|
| ETH | 0.707 | $1,414 |
| USDC | 1,414 | $1,414 |
| **總計** | - | **$2,828** |

**對比單純持有：**
| 策略 | ETH 價值 | USDC 價值 | 總價值 |
|------|---------|----------|--------|
| 流動性提供 | $1,414 | $1,414 | **$2,828** |
| 單純持有 | $2,000 | $1,000 | **$3,000** |
| **差額（無常損失）** | - | - | **-$172（5.7%）** |

## 無常損失對照表

| 價格變動幅度 | 無常損失 |
|-------------|---------|
| 1.25x（+25%）| 0.6% |
| 1.5x（+50%）| 2.0% |
| 2x（+100%）| 5.7% |
| 3x（+200%）| 13.4% |
| 4x（+300%）| 20.0% |
| 5x（+400%）| 25.5% |

**重要**：價格下跌也會產生無常損失，損失是雙向的。

## 為什麼還要提供流動性

✅ **手續費收入**
- 每筆交易收取 0.05% - 1% 手續費
- 交易量大的池子可以彌補無常損失

✅ **流動性挖礦獎勵**
- 協議代幣獎勵（如 UNI、SUSHI）
- 有時獎勵遠超過無常損失

✅ **價格穩定幣對**
- 穩定幣池（USDC/USDT）幾乎無無常損失
- 相關性高的資產對（如 ETH/stETH）損失較小

## 降低無常損失的方法

| 方法 | 說明 |
|------|------|
| **選擇穩定幣池** | USDC/USDT 等價格錨定池 |
| **選擇相關資產對** | ETH/stETH、WBTC/renBTC |
| **使用集中流動性** | Uniswap V3 可設定價格區間 |
| **選擇低波動幣種** | 避開新幣、小幣 |
| **長期持有** | 短期波動可能回歸 |

## 風險與注意事項

⚠️ **損失可能超過手續費**
- 劇烈價格波動時，無常損失可能很大
- 牛市中熱門幣對最容易受損

⚠️ **無常損失 + 代幣貶值**
- 如果池中的代幣本身價格下跌
- 損失會疊加：無常損失 + 資產貶值

⚠️ **「無常」可能變「永久」**
- 價格回不來 = 損失永久實現
- 不要期待所有損失都會消失

⚠️ **智能合約風險**
- 除了無常損失，還要考慮合約被駭風險
- 選擇經過審計的知名協議
