# 助記詞（Seed Phrase / Recovery Phrase）

## 定義

**助記詞 = 用普通人類語言表示的私鑰備份**

通常由 **12 或 24 個英文單詞**組成，遵循 BIP39 標準。助記詞可以：
- 生成所有相關的[[私鑰]]和錢包地址
- 在任何兼容的錢包中恢復資產
- 是資產恢復的唯一方式

```
12 詞助記詞範例：
abandon ability able about above absent absorb abstract absurd abuse access accident

24 詞助記詞範例：
abandon ability able about above absent absorb abstract absurd abuse access account accuse achieve acid acoustic acquire across act action actor actress actual adapt
```

## 助記詞 vs [[私鑰]]

| 項目 | 助記詞 | 私鑰 |
|------|--------|------|
| **形式** | 12/24 個英文單詞 | 64 位十六進制字符串 |
| **可讀性** | 🟢 人類易讀易記 | 🔴 難以抄寫 |
| **功能** | 生成多個私鑰（多帳戶）| 控制單一地址 |
| **備份難度** | 容易手寫備份 | 容易出錯 |

## 為什麼重要

✅ **資產恢復的唯一途徑**
- 手機丟了？用助記詞恢復
- 錢包 App 故障？用助記詞恢復
- 硬件錢包損壞？用助記詞在新設備恢復

✅ **跨平台兼容**
- 一組助記詞可在不同錢包間導入
- 不受特定廠商或 App 限制

✅ **多帳戶管理**
- 一組助記詞可生成無限個地址
- 方便管理不同用途的資產

## 助記詞生成標準

| 標準 | 詞數 | 安全性 |
|------|------|--------|
| BIP39 | 12 詞 | 🟡 基礎（128 位熵）|
| BIP39 | 24 詞 | 🟢 更高（256 位熵）|

**註**：24 詞比 12 詞更安全，但 12 詞已足夠應對暴力破解。

## 安全儲存方法

### 🟢 推薦做法
| 方法 | 說明 |
|------|------|
| **金屬助記詞板** | 防火防水防腐蝕，最推薦 |
| **手寫紙本** | 多份備份，存放在安全地點 |
| **分開儲存** | 將單詞分成幾部分存放 |

### 🔴 絕對禁止
| 方法 | 風險 |
|------|------|
| 手機截圖 | 可能自動備份到雲端 |
| 電腦存檔 | 可能被惡意軟件竊取 |
| 郵件/雲端 | 帳號被駭 = 資產被盜 |
| 社交軟件 | 永遠不要數位分享 |
| 腦記 | 可能遺忘或混淆順序 |

## 風險與注意事項

⚠️ **助記詞 = 你的資產**
- 任何人擁有助記詞 = 完全控制你的資產
- 沒有銀行或客服可以幫你「凍結」或「恢復"

⚠️ **順序很重要**
- 單詞順序錯誤 = 無法恢復
- 建議編號後再抄寫

⚠️ **測試恢復**
- 首次備份後，建議在新設備測試恢復
- 確認助記詞正確無誤

⚠️ **社交工程防範**
- 永遠不向任何人透露助記詞
- 沒有「官方驗證」需要助記詞這回事

## 進階安全策略

### 多簽錢包（Multisig）
- 需要多組助記詞才能操作
- 適合大額資產或團隊管理

### Shamir 秘密分享
- 將助記詞分成多份
- 需要達到指定份數才能恢復
- 如 3-of-5：5 份中任何 3 份可恢復
