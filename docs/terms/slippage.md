# 滑點（Slippage）

## 定義

**滑點 = 交易執行價格與預期價格之間的差異**

在加密貨幣交易中，由於市場波動或流動性不足，最終成交價格可能與下單時看到的價格不同。這種差異稱為滑點。

```
滑點計算：
預期價格：$100
實際成交價：$98
滑點 = ($100 - $98) / $100 = 2%
```

## 滑點的類型

| 類型 | 說明 | 影響 |
|------|------|------|
| **正滑點** | 成交價比預期更好 | 對你有利 |
| **負滑點** | 成交價比預期更差 | 對你不利 |

大多數情況下，交易者在 DEX 上遇到的是**負滑點**。

## 為什麼會發生滑點

### 1. 價格波動
- 區塊鏈交易需要時間確認
- 市場價格可能在這期間變動
- 高波動性幣種滑點更大

### 2. 流動性不足
- 流動性池中的資金有限
- 大額交易會顯著影響價格
- 遵循 AMM 的「價格曲線」機制

### 3. MEV 攻擊
- 機器人搶先交易（Front-running）
- 三明治攻擊（Sandwich attack）
- 導致用戶獲得更差的價格

## 滑點容忍度設定

交易時可以設定「最大可接受滑點」：

| 滑點設定 | 適用場景 | 風險 |
|---------|---------|------|
| 0.5% | 主流幣對（ETH/USDC）| 交易可能失敗 |
| 1-2% | 一般交易 | 平衡 |
| 3-5% | 小幣/低流動性幣種 | 可能承受較大損失 |
| >10% | 極低流動性代幣 | 極高風險 |

## 為什麼重要

✅ **成本控制**
- 高滑點會顯著增加交易成本
- 大額交易尤其需要關注

✅ **交易成功**
- 滑點設太低 → 交易可能失敗
- 滑點設太高 → 可能以極差價格成交

✅ **防範攻擊**
- 合理的滑點設定可減少 MEV 損失
- 使用抗 MEV 的 RPC 或協議

## 減少滑點的方法

### 交易前
| 方法 | 說明 |
|------|------|
| 分批交易 | 大額拆分成多筆小額 |
| 選擇高流動性池 | 使用 TVL 較大的流動性池 |
| 避開高峰時段 | 網絡擁堵時滑點更高 |

### 交易中
| 方法 | 說明 |
|------|------|
| 設定合理滑點 | 根據幣種流動性調整 |
| 使用限價單 | 某些 DEX 支援（如 1inch、CowSwap）|
| 開啟 MEV 保護 | 使用 Flashbots 等私有內存池 |

### 工具推薦
- **1inch**：自動尋找最佳路徑
- **CowSwap**：意圖交易，無 MEV
- **Paraswap**：聚合多個 DEX

## 風險與注意事項

⚠️ **無限滑點風險**
- 某些詐騙代幣會強制設定極高滑點
- 可能讓你以極差價格買入毫無價值的幣

⚠️ **價格影響（Price Impact）**
- 大額交易對流動性池的即時影響
- 不同於滑點，但可以預先計算

⚠️ **手續費混淆**
- 確認顯示的是「滑點」還是「手續費」
- 總成本 = 價格 × (1 + 滑點) + Gas 費
