# DEX（去中心化交易所）

## 定義

**DEX = Decentralized Exchange（去中心化交易所）**

不需要註冊、不需要信任中介，直接用錢包交易的交易所。

## DEX vs CEX

| 項目 | DEX | [[CEX]] |
|------|-----|---------|
| **控制方** | 智能合約 | 公司 |
| **資產託管** | 自己保管 | 交易所保管 |
| **KYC** | 不需要 | 需要 |
| **交易速度** | 受區塊鏈限制 | 即時 |
| **費用** | Gas 費 + 手續費 | 較低 |
| **功能** | 較基礎 | 完整 |
| **風險** | 合約漏洞 | 交易所倒閉/被駭 |

## 主要 DEX 類型

### 1️⃣ AMM（自動做市商）— 最流行

| 項目 | 說明 |
|------|------|
| **原理** | 流動性池代替訂單簿 |
| **定價** | 數學公式自動定價（x*y=k）|
| **代表** | Uniswap、Curve、PancakeSwap |
| **優點** | 無需對手方，隨時可交易 |
| **缺點** | [[滑點]]、[[無常損失]] |

### 2️⃣ 訂單簿 DEX

| 項目 | 說明 |
|------|------|
| **原理** | 傳統買賣報價撮合 |
| **代表** | dYdX、Hyperliquid、Serum |
| **優點** | 價格精確、適合大單 |
| **缺點** | 需要較高流動性 |

### 3️⃣ DEX 聚合器

| 項目 | 說明 |
|------|------|
| **原理** | 比價多個 DEX，找最優價格 |
| **代表** | 1inch、Matcha、Paraswap |
| **優點** | 最優價格、減少滑點 |

## 如何使用 DEX？

1. **連接錢包**：MetaMask、Phantom 等
2. **選擇代幣**：輸入要賣的代幣和要買的代幣
3. **確認價格**：查看匯率和滑點
4. **簽署交易**：在錢包確認
5. **等待完成**：交易上鏈，幾秒到幾分鐘

## DEX 的風險

⚠️ **智能合約風險**
- 合約有 bug 可能導致資金損失
- 即使是知名 DEX 也有風險

⚠️ **滑點（Slippage）**
- 實際成交價格與預期不同
- 大單或低流動性時更嚴重

⚠️ **無常損失（Impermanent Loss）**
- 提供流動性時可能發生
- 代幣價格變動導致損失

⚠️ **假代幣**
- 任何人都能創建代幣
- 要確認合約地址正確
