# Web3 vs Web2：具體差異

## 銀行轉帳的例子

| | **Web2（傳統銀行）** | **Web3（加密貨幣）** |
|---|---|---|
| **誰控制** | 銀行 | 你自己 |
| **開戶** | 需要身分證、地址證明 | 只需要下載 App |
| **轉帳時間** | 1-3 個工作日 | 幾分鐘到幾秒 |
| **手續費** | $10-50 USD | $0.01-50 USD（視網路） |
| **審查** | 銀行可以凍結帳戶 | 沒有人能凍結你的錢包 |
| **出錯怎麼辦** | 打客服、申訴 | **沒有客服，交易不可逆** |

## 社交平台的例子

| | **Web2（Twitter）** | **Web3（Lens Protocol）** |
|---|---|---|
| **帳號所有權** | Twitter 擁有 | 你擁有（NFT 形式） |
| **粉絲資料** | 鎖在平台內 | 帶著走，換平台粉絲還在 |
| **內容審查** | Twitter 決定什麼能發 | 協議層無法審查 |
| **廣告收入** | 平台賺走 | 可以設定創作者分潤 |

## Web3 的代價

Web3 聽起來很美好，但有好處就有代價：

### ✅ 優點
- 真正的資產所有權
- 抗審查
- 開放、無需許可
- 全球性、24/7 運作

### ⚠️ 缺點/風險
- **沒有客服** — 轉錯地址，錢就沒了
- **你要自己負責安全** — 被駭沒人賠
- **技術門檻** — 需要學習
- **詐騙很多** — 監管較少
- **波動極大** — 資產可能一夜歸零

## 關鍵心態轉換

| Web2 思維 | Web3 思維 |
|-----------|-----------|
| 「平台會保護我」 | 「我自己負責」 |
| 「忘記密碼可以重設」 | 「忘記私鑰，資產永遠消失」 |
| 「有問題找客服」 | 「Code is Law，交易不可逆」 |
| 「這家公司很穩」 | 「智能合約風險，隨時可能被駭」 |
